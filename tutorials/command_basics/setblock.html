Command Basics - /setblock

The /setblock command has 2, 3 or 4 paramaters. The first paramater is the position. Learn more about positions here. The second is the block you want to place. After that is the method. The method is optional, and can be 'keep', 'replace', 'strict' or 'destroy'. Let's go over each of the methods.


The replace method / No method:

/setblock ~ ~2 ~ minecraft:tuff_bricks

This places tuff bricks 2 blocks above the execution location regardless of what block is present there, and will cause block updates if certain blocks are already around it (eg. still water realising there is space to flow, stairs changing shape).


The keep method:

/setblock ~ ~2 ~ minecraft:tuff_bricks keep

Probably the most boring one of all of them. I'll say it now to get it over with. The keep method will place the tuff bricks only if air is in the target location. Any other block present, and it will fail to set the block. The block update rule from above applies here.


The strict method:

/setblock ~ ~2 ~ minecraft:tuff_bricks replace

This method is rather exciting for some datapack developers. The strict method acts the same as replace, except that it does not cause any block updates! This is a relatively new feature (since 1.21.5).


The destroy method:

/setblock ~ ~2 ~ minecraft:tuff_bricks destroy

Now for the final method, destroy. Another branch of the replace method, though the difference is that it breaks the block that was previously there, and drops the item of that block.
