Command Basics - /give

The give command has 3 paramaters - a target selector (the player to give the item to), an item stack (the item's ID and components), and an int (the item count). Only the first two paramaters are required. If the last paramater is ignored, the amount of items will be 1.


Here is an example:


/give @p minecraft:apple[] 1


Of course, that is a very basic example. You can modify the item's properties using the components in the item stack paramater.


/give @p minecraft:apple[ item_name= "Shiny Red Apple", enchantment_glint_override= true ] 12

"But hold on, I don't understand that! You just introduced me to the /give command and this comes up straight away!"


I totally get you. In Bedrock Edition, the /give command has components that work similar to block states. However, in Java Edition, we use components to not just, for example, force the item to stay in one inventory slot, but to apply certain characteristics to the item they are put on. Whilst I will leave the topic of item components to their own tutorial (see below), I can give you a teaser. In the above example, we give the apple a 'glowing' or 'enchanted' effect. This is acheived through the minecraft:enchantment_glint_override component. Every item component is provided its value through an equals (=) sign.
